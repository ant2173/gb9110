# Roadmap

## Phase 1 — prove the platform

- [x] Build a GEOS app with the Nokia SDK
- [x] Draw a static framebuffer
- [x] Run a periodic update
- [x] Adapt Peanut-GB to Borland C++ 4.52
- [x] Execute a real ROM
- [x] Render the first real frame
- [x] Add keyboard input
- [x] Run on real Nokia hardware

## Phase 2 — measure instead of guess

- [x] Separate CPU, PPU, packing, and blit costs
- [x] Remove full-window clearing and flicker
- [x] Replace accumulating continual timers with self-queued work
- [x] Test 4-bpp versus 1-bpp GEOS blits
- [x] Document the failed direct-packed renderer
- [x] Measure the lookup-table renderer: 12 FPS packed, 9 FPS full

## Phase 3 — make one ROM playable

- [ ] Bring the full path above 10 FPS (currently 9 FPS)
- [ ] Add adaptive frame skipping
- [ ] Decouple guest timing from display timing
- [ ] Verify input remains responsive under load
- [ ] Fix all visual differences against the original renderer
- [ ] Add an optional reduced-accuracy PPU mode

## Phase 4 — optimize the CPU core

- [ ] Profile opcode dispatch and memory access
- [ ] Remove avoidable far/near conversions
- [ ] Audit 32-bit arithmetic generated by Borland
- [ ] Build instruction-family benchmarks
- [ ] Evaluate a table-driven dispatcher
- [ ] Move only proven hotspots to 16-bit x86 assembly

## Phase 5 — turn the experiment into an emulator

- [ ] ROM chooser
- [ ] variable ROM sizes
- [ ] MBC behavior verified on hardware
- [ ] cartridge RAM
- [ ] save files
- [ ] pause/resume lifecycle
- [ ] configuration persistence
- [ ] compatibility list

## Deferred

- audio
- link cable
- Game Boy Color
- polished installer
- public binary release

Audio is deliberately deferred. A slow emulator with sound is still a slow emulator.
