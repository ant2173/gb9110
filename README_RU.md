# GB9110

**Экспериментальный эмулятор Game Boy для Nokia 9110 Communicator и PC/GEOS: разработка с помощью ИИ, проверка — на реальном железе.**

[English version](README.md)

![Настоящий ROM Game Boy запущен в оболочке Nokia 9110 SDK](docs/images/continuous-loop.jpg)

GB9110 — порт компактного ядра [Peanut-GB](https://github.com/deltabeard/Peanut-GB) в среду Nokia 9110 SDK и Borland C++ 4.52 конца 1990-х.

Проект давно прошёл стадию «hello world»:

- настоящий DMG-ROM загружается из файловой системы коммуникатора;
- адаптированное ядро LR35902 выполняется на реальной Nokia 9110;
- изображение Game Boy выводится в GEOS bitmap 160×144, 4 bpp;
- управление привязано к клавиатуре коммуникатора;
- CPU, PPU, упаковка пикселей и вывод GEOS измеряются раздельно;
- полный аппаратный путь ускорен с **5 до 13 FPS**.

Это всё ещё **не готовый универсальный эмулятор**. Сначала один небольшой открытый homebrew-ROM должен стать корректным и практически играбельным, затем можно расширять совместимость.

## Текущий результат на реальном аппарате

Лучшая измеренная сборка — `GBROW v1.3`. В ней объединены:

- упакованный регистр флагов и оптимизированные для 16-битного компилятора пути CPU;
- прямое чтение ROM и быстрый стек в WRAM;
- защищённая ROM-specific superinstruction для горячего цикла деления;
- фоновый рендерер ROW8 по восемь пикселей за итерацию;
- вывод GEOS 4-bpp.

| Путь | Результат на Nokia 9110 |
|---|---:|
| Первый полный оригинальный путь | 5 гостевых/экранных FPS |
| Табличный рендерер (`GBTABLE v0.6`) | 9 гостевых/экранных FPS |
| ROW8 (`GBROW v1.3`) | 12 гостевых/экранных FPS |
| DIV + ROW8 | **13 гостевых/экранных FPS** |
| Лучший CPU-only путь | 28 гостевых FPS |
| Статический GEOS blit 160×144 4-bpp | 44 FPS |

Полный путь стал **в 2,6 раза быстрее** первой измеренной аппаратной версии. Полной скорости Game Boy пока нет: существенное время теперь одновременно занимают CPU, рендеринг и передача кадра средствами GEOS.

## Профилирование на самой Nokia

Профилировщик работает прямо на коммуникаторе и считает инструкции, обращения к памяти и прерывания.

![Сводка аппаратного профиля на реальной Nokia 9110](docs/images/hardware-profile-summary.jpg)

За 180 гостевых кадров измерено:

```text
317 034 основных инструкций
 37 440 CB-инструкций
566 046 чтений памяти
 71 442 записи в память
  1 260 обслуженных прерываний
```

Десять самых частых opcode дают примерно половину потока инструкций. Четыре CB-команды вращения через перенос — около 95% всех CB-инструкций.

![Самые частые opcode LR35902 на экране Nokia 9110](docs/images/hardware-profile-opcodes.jpg)

Подробности: [HARDWARE_PROFILE.md](docs/HARDWARE_PROFILE.md) и [BENCHMARKS.md](BENCHMARKS.md).

## Что сработало, а что нет

Отрицательные результаты здесь не прячутся.

**Сработало:**

- табличный рендеринг по четыре пикселя: packed-путь `6 → 12 FPS`;
- первые CPU hot paths: core `22 → 25 FPS`;
- дальнейшая чистка CPU: core до `27 FPS`;
- superinstruction цикла деления: A/B `26 → 28 FPS`;
- ROW8: packed `13 → 17 FPS`, полный путь `10 → 12 FPS`;
- DIV + ROW8: полный путь `10 → 13 FPS`.

**Не сработало:**

- простое устранение отдельной упаковки framebuffer;
- большой кэш декодированных строк тайлов при 99,98–100% попаданий;
- общая чистка таймеров и housekeeping как источник крупного самостоятельного прироста.

Кэш тайлов замедлил полный путь с 10 до 8 FPS: стоимость самого hit-path оказалась выше компактного LUT-декодера.

## Среда сборки

Проверенная конфигурация:

- Nokia 9110 SDK (`N9110V10`);
- инструменты PC/GEOS;
- Borland C++ 4.52;
- `mkmf`, `pmake`, GOC и Glue.

Рабочие корни:

```text
C:\PCGEOS\N9110V10
C:\PCGEOS\User1
```

Сборка текущего эксперимента:

```bat
xcopy /E /I experiments\gbrow C:\PCGEOS\User1\Appl\GBROW
cd /d C:\PCGEOS\User1\Appl\GBROW
BUILD_GBROW.BAT
```

Текущая версия ожидает пользовательский тестовый ROM:

```text
FLAPPY.GB
```

ROM **не входит** в репозиторий. На реальном аппарате `GBROW.GEO` и ROM помещаются в:

```text
World\ExtrApps
```

На Nokia используется обычная `.GEO`-сборка, не EC. Подробности: [BUILDING.md](BUILDING.md).

## Как используется ИИ

Проект разрабатывается с помощью ИИ, но все содержательные утверждения проверяются на железе.

ИИ помогает изучать исходники, готовить патчи и диагностические сборки, вести документацию и сопоставлять профили. Человек устанавливает историческую среду, устраняет проблемы сборки, запускает каждую версию на настоящей Nokia 9110, проверяет картинку и управление и записывает измерения.

Правило простое: **оптимизация не считается успешной, пока это не подтвердил реальный аппарат.**

## Почему Nokia 9110

Это очень честная инженерная цель:

- настоящая многозадачная графическая ОС;
- сегментированная 16-битная среда C;
- строгие ограничения fixed data и code resources;
- широкий монохромный экран и полноценная клавиатура;
- старые инструменты, которые показывают цену решений, незаметную на современных системах.

Это ностальгия с измерительными приборами: архитектура эмулятора, память GEOS, событийное планирование, переносимость старого C и оптимизация в жёстких ограничениях.

## Ближайший план

Следующая практическая цель — отделить скорость гостевой системы от частоты вывода:

- фиксированный и адаптивный frameskip;
- независимые счётчики guest FPS и display FPS;
- проверка отзывчивости управления при пропуске кадров;
- облегчённая текущая сборка без исторических benchmark-режимов;
- затем новый profiler-driven эксперимент с CPU или ассемблером.

Поддержка разных картриджей, Save RAM, выбор ROM и звук — позже. См. [ROADMAP.md](ROADMAP.md).

## Статус проекта

Репозиторий следует воспринимать как **открытый инженерный журнал с запускаемыми исходниками**, а не как релиз готового эмулятора.

Сейчас особенно полезны знания в областях:

- 16-битный x86 и кодогенерация Borland C++ 4.x;
- память, графика и code resources PC/GEOS;
- dispatch и тайминги LR35902;
- оптимизация PPU Game Boy;
- разработка под Nokia Communicator.
