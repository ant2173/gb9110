# GB9110

**Экспериментальный эмулятор Game Boy для Nokia 9110 Communicator и PC/GEOS: разработка с помощью ИИ, проверка — на реальном железе.**

[English version](README.md)

![Настоящий ROM Game Boy работает на Nokia 9110](docs/images/div2-fs2-v2.jpg)

GB9110 — порт компактного ядра [Peanut-GB](https://github.com/deltabeard/Peanut-GB) в среду Nokia 9110 SDK и Borland C++ 4.52 конца 1990-х.

Проект давно прошёл стадию proof of concept:

- настоящий DMG-ROM загружается из файловой системы коммуникатора;
- адаптированное ядро LR35902 выполняется на реальной Nokia 9110;
- изображение Game Boy выводится в GEOS bitmap 160×144, 4 bpp;
- управление привязано к клавиатуре коммуникатора;
- CPU, рендеринг, упаковка, GEOS blit, opcode и обращения к памяти измерены на железе;
- полный путь с выводом каждого кадра ускорен с **5 до 13 FPS**;
- фиксированный frame skip даёт **18 гостевых FPS / 9 экранных FPS**;
- текущий CPU-only путь достигает **28 гостевых FPS**.

Это всё ещё **не готовый универсальный эмулятор**. Сначала один небольшой открытый homebrew-ROM должен стать корректным и практически играбельным, затем можно расширять совместимость.

## Текущий результат на реальном аппарате

| Путь | Результат на Nokia 9110 |
|---|---:|
| Первый полный путь | 5 гостевых/экранных FPS |
| GBTABLE v0.6 | 9 гостевых/экранных FPS |
| GBROW v1.3 | 12 гостевых/экранных FPS |
| GBROW v1.3 + DIV v1 | **13 гостевых/экранных FPS** |
| GBFS v1.4c, FS2 | **18 гостевых / 9 экранных FPS** |
| GBFS v1.4c, FS3 | 21 гостевой / 7 экранных FPS |
| GBDIV2 v1.6, CPU only | **28 гостевых FPS** |
| Статический GEOS blit 160×144 4-bpp | 44 FPS |

Frame skip не пропускает гостевое выполнение: CPU, таймеры, управление и логика игры идут каждый гостевой кадр; на выбранных кадрах вместе пропускаются рендеринг и GEOS blit.

## Текущий профилировщик

`GBPROF v1.5j` измеряет уже актуальную нагрузку ROW8/DIV прямо на коммуникаторе. Байты выборки opcode исключены из профиля областей памяти, чтобы отдельно видеть обращения к данным.

## Результат DIV v2

Первый DIV-helper сворачивал полную итерацию ROM-specific цикла деления, только когда безопасно помещалось длинное окно гостевых тактов. DIV v2 сохраняет этот путь и добавляет два коротких защищённых блока:

- `PREFIX`: `PUSH AF + RL L/H/C/B`, 48 гостевых тактов;
- `FINAL`: фиксированный эпилог деления, 52 гостевых такта.

В CPU-only замере fallback снизился с **929/с до 350/с**. В FS2 — с **619/с до 229/с**. CPU вырос с 26 FPS без helper до 27 FPS с DIV v1 и 28 FPS с DIV v2.

![DIV v2 в CPU benchmark на Nokia 9110](docs/images/div2-core-v2.jpg)

Это намеренно ROM-specific оптимизация с точной проверкой сигнатуры и границ тайминга, а не универсальная функция эмулятора.

## Что сработало, а что нет

**Сработало:**

- табличный рендеринг по четыре пикселя: packed-путь `6 → 12 FPS`;
- CPU hot paths: core `22 → 25 FPS`;
- дальнейшая чистка CPU: core до `27 FPS`;
- DIV v1: A/B core `26 → 28 FPS` в исходном benchmark;
- ROW8: packed `13 → 17 FPS`, полный путь `10 → 12 FPS`;
- DIV + ROW8: полный путь `10 → 13 FPS`;
- FS2: `13/13 → 18/9` гостевых/экранных FPS;
- DIV v2: fallback уменьшился примерно на 62% в обоих сравнительных замерах.

**Не сработало:**

- простое устранение отдельной упаковки framebuffer;
- большой кэш декодированных строк тайлов при 99,98–100% попаданий;
- общая чистка таймеров и housekeeping как источник крупного самостоятельного прироста.

Кэш тайлов замедлил полный путь с 10 до 8 FPS: стоимость hit-path оказалась выше компактного LUT-декодера.

## Среда сборки

Проверенная конфигурация:

- Nokia 9110 SDK (`N9110V10`);
- инструменты PC/GEOS;
- Borland C++ 4.52;
- `mkmf`, `pmake`, GOC и Glue.

Рабочие корни:

```text
C:\PCGEOS\N9110V10
C:\PCGEOS\User1
```

Сборка текущего benchmark:

```bat
xcopy /E /I experiments\gbdiv2 C:\PCGEOS\User1\Appl\GBDIV2
cd /d C:\PCGEOS\User1\Appl\GBDIV2
BUILD_GBDIV2.BAT
```

Эксперименты ожидают пользовательский ROM `FLAPPY.GB`; ROM **не входит** в репозиторий. Обычный `.GEO` и ROM помещаются на Nokia в `World\ExtrApps`. Подробности: [BUILDING.md](BUILDING.md).

## Как используется ИИ

Проект разрабатывается с помощью ИИ, но все содержательные утверждения проверяются на железе. ИИ помогает изучать исходники, готовить патчи и диагностические сборки, вести документацию и сопоставлять профили. Человек устанавливает историческую среду, устраняет проблемы сборки, запускает каждую версию на настоящей Nokia 9110, проверяет картинку и управление и записывает измерения.

Правило простое: **оптимизация не считается успешной, пока это не подтвердил реальный аппарат.**

## Следующий этап: точечный 8086 asm

Очередная широкая перепись на C уже вряд ли даст лучший результат за потраченное время. Следующий эксперимент — небольшой обратимый fast path на 16-битном x86 вокруг выбранных профилировщиком операций LR35902 и накладных расходов dispatch/памяти.

Первая asm-сборка сохранит:

- C-интерпретатор как эталон и fallback;
- DIV v2 и FS2 как текущую базу;
- A/B режимы в одном бинарнике;
- проверку состояния и ошибок;
- узкий объём изменений вместо переписывания всего CPU.

См. [ASM_PLAN.md](docs/ASM_PLAN.md) и [ROADMAP.md](ROADMAP.md).
